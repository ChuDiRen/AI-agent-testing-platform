# 命令：analyze-requirement

## 功能描述

深入分析产品需求，输出结构化的需求文档，包括功能分析、用户场景、技术挑战等。

## 使用方式

```
/analyze-requirement
```

或

```
/analyze-requirement <需求描述>
```

## 执行流程

1. **收集需求信息**：
   - 项目名称和类型
   - 核心功能列表
   - 目标用户群体
   - 预期用户规模

2. **业务分析**：
   - 业务领域识别
   - 业务模式分析（B2B/B2C/O2O等）
   - 核心价值主张

3. **用户分析**：
   - 用户画像
   - 用户路径分析
   - 使用场景梳理

4. **功能拆解**：
   - 功能模块划分
   - 功能优先级评估
   - 功能依赖关系

5. **技术挑战识别**：
   - 高并发场景
   - 大数据处理
   - 实时性要求
   - 数据一致性要求

6. **输出文档**：生成结构化的需求文档

## 输出格式

```markdown
【需求分析报告】

## 项目概述
- 项目名称：xxx
- 项目类型：电商/管理/移动H5
- 核心价值：xxx

## 功能模块
### 模块1：用户管理
- 用户注册/登录
- 个人信息管理
- 权限控制

### 模块2：商品管理
- 商品列表
- 商品详情
- 搜索筛选

...

## 用户场景
### 场景1：用户浏览商品
用户打开APP -> 浏览商品列表 -> 查看商品详情 -> 加入购物车

...

## 技术挑战
1. 高并发：预期QPS达到1000+
2. 数据一致性：订单状态同步
3. 实时性要求：库存实时更新

## 技术选型建议
- 后端：Spring Boot（企业级稳定性）
- 前端：Vue3 + Element Plus
- 数据库：PostgreSQL + Redis

## 下一步
输入 "确认" 进入原型设计阶段
```

## 示例

```
/analyze-requirement 我需要做一个移动端H5商城，用户可以查看商品列表、加入购物车、浏览购物车
```
