# 项目启动

启动新项目：$ARGUMENTS

## 当前状态检查

- 工作目录：!pwd
- 现有项目结构：!ls -la

## 任务

调用 `project-bootstrapper` Agent 完成项目启动的完整流程。

### 流程概览

```
[1] 初始化 → [2] 需求 → [3] 原型 → [4] 任务拆分 → [5] API设计 → [6] 开发准备
```

### 阶段输出物

| 阶段 | 输出物 | 存放位置 |
|-----|-------|---------|
| 初始化 | 项目结构、配置文件 | 根目录 |
| 需求 | 产品需求文档 | `doc/PRD.md` |
| 原型 | HTML 原型页面 | `prototype/` |
| 任务拆分 | 前后端任务清单 | `doc/frontend-tasks.md`、`doc/backend-tasks.md` |
| API设计 | 接口文档 | `doc/api/*.md` |
| 开发准备 | 功能清单 | `doc/func.md` |

### 开发模式选择

| 项目规模 | 推荐模式 | 目录结构 |
|---------|---------|---------|
| 小型 | 统一管理 | `project/frontend/` + `project/backend/` |
| 中大型 | 分离管理 | 独立仓库 |
