# 原型设计 Skill

## 概述
本 Skill 用于根据需求文档生成高保真的原型界面，支持移动端和 PC 端应用。整合了 UI/UX Pro Max 设计智能库的最佳实践。

**参考来源**: [UI/UX Pro Max Skill](https://github.com/nextlevelbuilder/ui-ux-pro-max-skill)

## 使用场景
- 快速出原型与老板/客户确认需求
- 为正式页面开发做准备
- UI/UX 设计验证

## 原型设计提示词模板

### 移动端 H5 原型
```prompt
你是一个专业的UI设计师，你需要根据我提供的需求文档来完成页面的设计

请仔细阅读需求文档 @[需求文档路径]，现在需要输出高保真的原型图，请通过以下方式帮我完成所有界面的原型设计，并确保这些原型界面可以直接用于开发：

1、用户体验分析：先分析这个 App 的主要功能和用户需求，确定核心交互逻辑。

2、产品界面规划：作为产品经理，定义关键界面，确保信息架构合理。

3、高保真 UI 设计：作为 UI 设计师，设计贴近真实 iOS/Android 设计规范的界面，使用现代化的 UI 元素，使其具有良好的视觉体验。

4、HTML 原型实现：使用 HTML + Tailwind CSS（或 Bootstrap）生成所有原型界面，并使用 FontAwesome（或其他开源 UI 组件）让界面更加精美、接近真实的 App 设计。拆分代码文件，保持结构清晰：

5、每个界面应作为独立的 HTML 文件存放，例如 home.html、profile.html、settings.html 等。

• index.html 作为主入口，不直接写入所有界面的 HTML 代码，而是使用 iframe 的方式嵌入这些 HTML 片段，并将所有页面直接平铺展示在 index 页面中，而不是跳转链接。

• 真实感增强：
  - 界面尺寸应模拟 iPhone 15 Pro，并让界面圆角化，使其更像真实的手机界面。
  - 使用真实的 UI 图片，而非占位符图片（可从 Unsplash、Pexels、Apple 官方 UI 资源中选择）。
  - 添加顶部状态栏（模拟 iOS 状态栏），并包含 App 导航栏（类似 iOS 底部 Tab Bar）。

请按照以上要求生成完整的 HTML 代码，并确保其可用于实际开发
```

## UX 设计指南（98条精选）

### 导航
- **平滑滚动**: 使用 `scroll-behavior: smooth` 实现锚点平滑滚动
- **固定导航栏**: 添加与导航栏高度匹配的 `padding-top`，避免遮挡内容
- **激活状态**: 高亮当前页面/章节的导航项
- **面包屑**: 适用于层级较深的网站（3层以上）

### 动画
- **避免过度动画**: 每屏仅动画化 1-2 个关键元素
- **动画时长**: 微交互建议 150-300ms，避免超过 500ms
- **减少动画偏好**: 通过 `prefers-reduced-motion` 媒体查询适配

### 布局
- **z-index 管理**: 使用系统化层级（如 10/20/50），避免随意设置超大值
- **布局偏移**: 为异步加载内容预留空间（如固定宽高比）
- **视口单位**: 移动端避免直接使用 `100vh`，推荐 `dvh` 或 `min-h-screen`

### 触摸交互
- **触摸目标尺寸**: 最小 44×44px，确保易点击
- **触摸间距**: 相邻可点击元素间距至少 8px
- **点击延迟优化**: 使用 `touch-action: manipulation` 减少 300ms 延迟

### 可访问性
- **颜色对比度**: 文本与背景对比度至少 4.5:1
- **非颜色依赖**: 使用图标或文字辅助颜色传递信息
- **Alt 文本**: 为内容性图片添加描述性 `alt` 属性
- **键盘导航**: 确保所有功能可通过键盘访问

### 表单
- **输入框标签**: 始终使用可见的 `<label>`，避免仅依赖占位符
- **错误提示位置**: 在输入框附近显示错误信息
- **实时验证**: 在输入失焦时进行验证
- **密码可见性**: 提供显示/隐藏密码的切换按钮

### 响应式设计
- **移动优先**: 先设计移动端样式，再通过断点扩展
- **字体可读性**: 移动端正文文本最小 16px
- **表格处理**: 移动端为表格添加横向滚动或卡片布局

### 反馈
- **加载指示器**: 操作超过 300ms 时显示加载状态
- **空状态提示**: 引导用户操作，避免空白页面
- **错误恢复**: 提供明确的下一步操作建议

## 专业 UI 设计规则

### 图标与视觉元素
- ❌ 禁止使用表情符号作为图标（使用专业图标库）
- ✅ 使用 Lucide、Heroicons、FontAwesome 等专业图标
- ✅ 确保图标大小一致（推荐 20px/24px）
- ✅ 悬停状态稳定，避免布局抖动

### 交互与光标
- ✅ 可点击元素需有指针光标 (`cursor: pointer`)
- ✅ 提供悬停反馈（颜色变化、阴影等）
- ✅ 禁用状态使用 `cursor: not-allowed`

### 明暗模式
- ✅ 深色模式背景不要纯黑，推荐 `#0F172A` 或 `#1E293B`
- ✅ 确保文字与背景对比度符合 WCAG 标准
- ✅ 使用 CSS 变量管理主题色

### 布局与间距
- ✅ 使用 8px 网格系统
- ✅ 内边距和外边距保持一致
- ✅ 避免内容过于紧凑

## 工作流程

1. **阅读需求文档**：理解核心功能和用户需求
2. **选择设计风格**：根据产品类型选择合适的 UI 样式
3. **确定配色方案**：根据行业选择配色
4. **选择字体配对**：根据风格选择字体组合
5. **规划页面结构**：列出所有需要的页面
6. **设计核心页面**：先完成主要功能页面
7. **完善细节**：添加交互效果和动画
8. **局部调整**：根据反馈进行修改
9. **确认交付**：与老板/客户确认需求
