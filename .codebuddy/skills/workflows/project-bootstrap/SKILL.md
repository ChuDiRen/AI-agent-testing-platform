# 项目启动工作流

## 概述
本 Skill 提供从零开始启动一个完整项目的标准化工作流。

## 适用场景
- 新项目启动
- 功能模块重构
- 独立子系统开发

---

## 核心原则（必须遵守）

1. **每个阶段必须完成所有输出物并通过检查点验证后，才能进入下一阶段**
2. **禁止跳过任何阶段或输出物**
3. **禁止在文档中标记"✅完成"但实际未生成文件**
4. **每个阶段结束时必须执行检查点验证**

---

## 工作流阶段

```
阶段0: 初始化 → 阶段1: 需求分析 → 阶段2: 原型设计 → 阶段3: 任务拆分 → 阶段4: API设计 → 阶段5: 开发准备
```

### 阶段0: 项目初始化
**输出物**:
- `doc/` 目录结构
- `func.md` 功能清单文件

### 阶段1: 需求分析
**输出物**:
- `doc/PRD.md` 产品需求文档

**PRD 必须包含**:
- 项目概述
- 功能需求
- 页面清单
- 数据模型
- 非功能需求

### 阶段2: 原型设计
**输出物**:
- `prototype/` 原型文件目录
- 所有页面的 HTML 文件

### 阶段3: 任务拆分
**输出物**:
- `doc/tasks/` 任务文档目录
- 前端任务清单
- 后端任务清单

### 阶段4: API 设计
**输出物**:
- `doc/api/` API 文档目录
- 各模块 API 文档

### 阶段5: 开发准备
**输出物**:
- 技术选型文档
- 开发环境配置指南

---

## 检查点验证

每个阶段结束时必须验证：
```markdown
## 阶段 X 检查点

### 输出物清单
- [ ] 文件1: `path/to/file1`
- [ ] 文件2: `path/to/file2`

### 验证结果
- 所有文件已生成: ✅/❌
- 内容完整性: ✅/❌

### 下一步
[进入下一阶段 / 补充缺失内容]
```
