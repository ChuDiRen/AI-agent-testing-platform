# 项目启动工作流 Skill

## 概述
本 Skill 提供从零开始启动一个完整项目的标准化工作流，自动引导完成从需求分析到开发准备的全过程。适用于任何技术栈。

## 适用场景
- 新项目启动
- 功能模块重构
- 独立子系统开发

---

## 使用方式

### 快速启动
```prompt
使用 project-bootstrap skill 启动项目：[项目名称]

项目类型：[管理后台/移动端H5/小程序/API服务]
核心功能：
1. [功能1]
2. [功能2]
3. [功能3]

技术栈：
- 前端：[框架名称]
- 后端：[框架名称]
- 数据库：[数据库名称]
```

## 阶段1：项目初始化

### 输入信息收集
```prompt
请提供以下信息：
1. 项目名称：
2. 项目类型：[管理后台/移动端H5/小程序/API服务]
3. 项目规模：[小型/中型/大型]
4. 技术栈选择：
   - 前端框架：[可选]
   - UI组件库：[可选]
   - 后端框架：[可选]
   - 数据库：[可选]
```

### 开发模式选择

| 项目规模 | 推荐模式 | 目录结构 |
|---------|---------|---------|
| 小型 | 统一管理 | `project/frontend/` + `project/backend/` |
| 中大型 | 分离管理 | 独立仓库，不同编辑器窗口 |

### 输出物
- [ ] 项目目录结构
- [ ] 基础配置文件
- [ ] 依赖安装完成

### 完成标志
```
✅ 前端项目正常启动
✅ 后端服务正常启动
```

## 阶段2：需求分析

### 需求文档生成提示词
```prompt
我现在需要做一个 {项目类型}，项目名称：{项目名称}

产品的功能包括：
1. {功能1}
2. {功能2}
3. {功能3}
...

请帮我完成一个产品需求文档，需要包含：

## 1. 项目概述
- 项目背景
- 目标用户
- 核心价值

## 2. 功能模块
- 模块划分
- 功能描述
- 优先级排序

## 3. 页面设计
- 页面清单
- 页面元素
- 交互说明

## 4. 数据模型
- 实体关系
- 表结构设计
- 字段说明

## 5. 非功能需求
- 性能要求
- 安全要求
- 兼容性要求

注意：不要包含任何代码
```

### 输出物
- [ ] 产品需求文档 (`doc/PRD.md`)
- [ ] 功能模块清单
- [ ] 数据模型设计

### 完成标志
```
✅ 需求文档已确认，无歧义
✅ 数据模型已评审
```

## 阶段3：原型设计

### 调用 prototype-design skill
```prompt
@prototype-design 

请根据需求文档 @doc/PRD.md 生成高保真原型界面：

项目类型：{管理后台/移动端H5}
页面清单：
1. {页面1}
2. {页面2}
3. {页面3}
...
```

### 原型文件结构
```
prototype/
├── index.html          # 主入口（平铺展示所有页面）
├── {page1}.html        # 页面1
├── {page2}.html        # 页面2
├── css/
│   └── styles.css
└── js/
    └── main.js
```

### 输出物
- [ ] 原型页面 HTML 文件
- [ ] 页面交互说明

### 完成标志
```
✅ 原型已与产品/客户确认
✅ 页面流程无遗漏
```

## 后端任务拆分
- 技术栈：{后端框架}

- {注意点2}

- [ ] 前端任务清单 (`doc/frontend-tasks.md`)
- [ ] 后端任务清单 (`doc/backend-tasks.md`)

## 阶段5：API 接口设计

### 调用 api-documentation skill
```prompt
@api-documentation

请根据需求文档和任务清单，设计完整的 API 接口规范：

## 设计要求
1. 列出所有需要的接口
2. 按模块分组
3. 统一响应格式
4. 定义 Mock 数据结构（JSON 格式，写在文档中）

## 输出格式
- 接口文档：doc/api/{module}-api.md（包含请求/响应示例）
```

### 统一响应格式
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 分页响应格式
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "items": [],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 输出物
- [ ] API 接口文档 (`doc/api/*.md`)
- [ ] 错误码定义

### 完成标志
```
✅ 接口文档已由前后端共同确认
✅ 响应数据结构定义完整
✅ 错误码定义完整
```

## 阶段6：开发准备

### 创建功能清单文档
```markdown
# 功能服务清单

## 项目信息
- 项目名称：{project-name}
- 创建时间：{date}
- 技术栈：{tech-stack}

## 前端服务

### API 层
| 文件 | 功能 | 方法 |
|-----|------|-----|
| api/auth | 认证模块 | login, logout, getUserInfo |
| api/user | 用户管理 | getList, create, update, delete |

### Store 层
| 文件 | 功能 | 状态 |
|-----|------|-----|
| stores/user | 用户状态 | userInfo, token, permissions |

### 组件层
| 文件 | 功能 | 复用场景 |
|-----|------|---------|
| components/TablePage | 通用表格页 | 列表页面 |

## 后端服务

### Service 层
| 文件 | 功能 | 方法 |
|-----|------|-----|
| services/user_service | 用户服务 | get_list, create, update, delete |

### Repository 层
| 文件 | 功能 | 方法 |
|-----|------|-----|
| repositories/user_repo | 用户数据访问 | get_by_id, get_by_username |
```

### 开发规范确认
```markdown
## 开发规范检查清单

### 前端规范
- [ ] 已阅读 frontend-development skill
- [ ] 已配置代码格式化工具
- [ ] 已配置 Mock 数据管理
- [ ] 已创建 API 封装

### 后端规范

- [ ] 已配置分层架构
- [ ] 已创建数据库迁移
- [ ] 已配置异常处理

### 代码复用
- [ ] 已创建功能清单文档
- [ ] 已确认无重复开发
```

### 输出物
- [ ] 功能清单文档
- [ ] 开发规范确认
- [ ] 开发环境就绪

### 完成标志
```
✅ 所有准备工作完成
✅ 可以开始执行开发任务
```

## 后续流程

完成 project-bootstrap 后，进入开发阶段：

```
project-bootstrap 完成
        ↓
┌───────┴───────┐
↓               ↓
前端开发         后端开发
↓               ↓
└───────┬───────┘
        ↓
    前后端联调
        ↓
    测试部署
```

### 开发阶段提示词
```prompt
@frontend-development 执行前端任务 TASK-FE-001

@backend-development 执行后端任务 TASK-BE-001
```

### 联调阶段提示词
```prompt
后端服务地址：http://xxx:8000
请关闭 Mock 模式，开始前后端联调
```

### 测试阶段提示词
```prompt
@api-testing 执行接口自动化测试

@webapp-testing 执行端到端测试
```
