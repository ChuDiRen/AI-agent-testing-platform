# 任务拆分 Skill

## 概述
本 Skill 用于将产品需求拆分为可执行的开发任务，确保开发过程有序进行。适用于任何技术栈的前后端项目。

## 核心原则
> **跟 AI 对话一定是任务越小越细，返回的代码质量就会越高，反之，大概率会返回有问题的代码。**

---

## 前端任务拆分

### 拆分提示词模板
```prompt
你是具有丰富开发经验的前端开发工程师，请你阅读 @[需求文档] 这个产品及UI文档，你需要完成这个产品的前端页面开发，请你按照你丰富的前端开发经验，拆解出详细的前端开发任务，按照以下格式保存到 doc 目录中。

- 根据开发顺序创建 TASK-FE-001 这样的任务编号；
- 每个任务包含名称，任务描述，版本，状态（计划中、进行中、已完成等）。
- 每个 TASK 都有验收标准清单和注意事项
```

### 前端任务模板
```markdown
# 前端开发任务清单

## TASK-FE-001 - 项目初始化与基础配置

**任务名称：** 项目初始化与基础配置
**任务描述：** 初始化前端项目，配置基础开发环境，包括路由、状态管理、API 封装等
**预估工时：** 4 小时
**状态：** 计划中

### 验收标准
- [ ] 项目能够正常启动
- [ ] 路由配置完成
- [ ] 状态管理配置完成
- [ ] API 请求封装完成
- [ ] Mock 数据管理配置完成

### 注意事项
- 配置代理支持
- 确保支持本地 Mock 和远程 API 切换

---

## TASK-FE-002 - 布局与导航

**任务名称：** 布局与导航
**任务描述：** 完成整体布局，包括导航栏、侧边栏、主内容区域
**预估工时：** 4 小时
**状态：** 计划中

### 验收标准
- [ ] 导航栏显示正常
- [ ] 侧边栏/底部栏切换正常
- [ ] 页面布局适配
- [ ] 响应式设计完成

### 注意事项
- 注意安全区域适配
- 导航切换需要保持页面状态

---

## TASK-FE-003 - 列表页面

**任务名称：** 列表页面开发
**任务描述：** 完成列表页面，包括搜索、筛选、分页、操作功能
**预估工时：** 6 小时
**状态：** 计划中

### 验收标准
- [ ] 列表正常显示
- [ ] 搜索功能正常
- [ ] 筛选功能正常
- [ ] 分页功能正常
- [ ] 新增/编辑/删除功能正常
- [ ] 空状态显示正常

### 注意事项
- 搜索需要防抖处理
- 大数据量考虑虚拟滚动
```

## 后端任务拆分

### 拆分提示词模板
```prompt
你是具有丰富开发经验的后端开发工程师，请阅读以下的需求和 API 文档，你需要完成这个产品的后端开发，请你按照你丰富的后端开发经验，拆解出详细的后端开发任务，按照以下格式保存到 doc 目录中的任务 md 文件中。

格式如下：
• 根据开发顺序创建 TASK-BE-001 这样的任务编号；
• 每个任务包含名称，任务描述，预估工时，状态（计划中、进行中、已完成等）。
• 每个 TASK 都有验收标准清单和注意事项
• 任务描述中不需要返回任何代码示例
```

### 后端任务模板
```markdown
# 后端开发任务清单

## TASK-BE-001 - 数据库设计与初始化

**任务名称：** 数据库设计与初始化
**任务描述：** 设计数据库表结构，创建数据库迁移脚本，初始化基础数据

- [ ] 迁移脚本创建完成
- [ ] 基础数据初始化完成
- [ ] 索引优化完成

### 注意事项
- 表名使用下划线命名

- 外键关系需要明确定义
- 需要考虑数据量增长的索引优化
```

## 任务执行提示词

### 执行单个任务
```prompt
@任务文档.md 现在你开始执行任务，每次只能执行一个任务，执行任务完成后需要更新任务状态以及验收清单。功能和接口规范需要跟 @doc 下文档保持一致。必须是我回复后才能继续下一项任务
```

### 继续下一个任务
```prompt
继续执行下一个任务，更新任务状态
```

## 任务状态管理

### 状态定义
| 状态 | 说明 |
|------|------|
| 计划中 | 任务已创建，等待开始 |
| 进行中 | 正在进行开发 |
| 测试中 | 开发完成，正在测试 |
| ✅ 已完成 | 任务已完成并验收通过 |
| 暂停 | 任务暂时搁置 |
| 取消 | 任务已取消 |

### 状态更新规范
```markdown
## TASK-BE-003 - 业务模块

**状态：** ~~计划中~~ → 进行中

### 验收标准
- [x] 列表查询接口完成
- [x] 详情查询接口完成
- [ ] 创建接口完成
- [ ] 更新接口完成
- [ ] 删除接口完成
- [ ] 分页筛选功能完成

### 开发记录
- 2025-01-21: 完成列表和详情接口
```

## 任务拆分最佳实践

### 拆分粒度
- 每个任务应该在 2-8 小时内完成
- 任务之间应该有明确的边界
- 避免任务之间的循环依赖

### 任务排序
1. 基础设施和配置优先
2. 核心功能模块优先
3. 依赖关系决定顺序
4. 复杂功能拆分为多个子任务

### 验收标准
- 每个验收项应该是可验证的
- 使用清晰的动词描述（完成、实现、支持）
- 避免模糊的描述

### 注意事项
- 记录可能的坑和陷阱
- 记录需要注意的技术细节
- 记录与其他任务的关联
