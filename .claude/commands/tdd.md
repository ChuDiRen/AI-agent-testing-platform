# TDD Command

## Instructions

1.  **Understand the Goal**
    - Analyze the feature description provided in arguments.
    - Identify the specific behavior to test.

2.  **Create Test (Red)**
    - Locate the appropriate test file or create a new one.
    - Write a test case that asserts the expected behavior.
    - Run the test to confirm it fails.

3.  **Implement (Green)**
    - Write the minimal implementation code.
    - Run the test again to confirm it passes.

4.  **Refactor (Refactor)**
    - Review the code for clarity and duplication.
    - Optimize if necessary.
    - Ensure tests still pass.

5.  **Repeat**
    - Move to the next requirement.
