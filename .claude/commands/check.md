# /check - 全栈代码规范检查

## 描述
对项目代码进行全面的规范检查，输出详细的检查报告。

## 使用方式
```
/check [--scope <范围>] [--fix]
```

## 参数说明
- `--scope`: 检查范围，可选值：`all`、`backend`、`frontend`、`changed`
- `--fix`: 自动修复可修复的问题

## 检查项目

### 后端检查 (Python)
| 检查项 | 说明 | 工具 |
|--------|------|------|
| 代码格式 | PEP8 规范 | ruff/black |
| 类型检查 | 类型注解 | mypy |
| 导入排序 | import 顺序 | isort |
| 安全检查 | 安全漏洞 | bandit |
| 复杂度 | 圈复杂度 | radon |

### 前端检查 (TypeScript/Vue)
| 检查项 | 说明 | 工具 |
|--------|------|------|
| 代码格式 | 代码风格 | ESLint/Prettier |
| 类型检查 | TypeScript | tsc |
| Vue 规范 | Vue 最佳实践 | eslint-plugin-vue |
| 样式检查 | CSS 规范 | Stylelint |

### 通用检查
| 检查项 | 说明 |
|--------|------|
| 命名规范 | 变量、函数、类命名 |
| 注释规范 | 必要的注释 |
| 文件结构 | 目录和文件组织 |
| Git 规范 | 提交信息格式 |

## 严重级别
- 🔴 **错误**: 必须修复，影响代码质量或功能
- 🟡 **警告**: 建议修复，影响可维护性
- 🔵 **提示**: 可选修复，代码优化建议

## 自动修复
使用 `--fix` 参数可自动修复以下问题：
- 代码格式问题
- 导入排序
- 简单的类型问题
- 未使用的导入
