# /dev - 智能开发流程

## 描述
7 步智能开发流程，从需求到交付的完整工作流。

## 使用方式
```
/dev <需求描述>
```

## 执行流程

### 步骤 1：需求澄清
1. 分析用户需求，提取核心功能点
2. 识别潜在的技术难点
3. 列出需要确认的问题
4. **等待用户确认后继续**

### 步骤 2：技术设计
1. 确定技术方案
2. 设计数据模型
3. 规划 API 接口
4. 输出设计文档

### 步骤 3：数据库设计
1. 设计表结构
2. 定义字段和索引
3. 生成 DDL 脚本
4. 考虑数据迁移

### 步骤 4：后端开发
1. 创建 Model/Entity
2. 创建 Repository/DAO
3. 创建 Service
4. 创建 Controller
5. 编写单元测试

### 步骤 5：前端开发
1. 创建 API 接口封装
2. 创建页面组件
3. 实现业务逻辑
4. 添加表单校验

### 步骤 6：测试验证
1. 运行单元测试
2. 进行接口测试
3. 检查代码规范
4. 修复发现的问题

### 步骤 7：文档更新
1. 更新 API 文档
2. 更新功能清单
3. 更新变更日志

## 输出物
- 后端代码（Model、Service、Controller）
- 前端代码（API、页面组件）
- 测试用例
- API 文档

## 注意事项
- 每个步骤完成后会显示进度
- 遇到问题会暂停并询问
- 可随时输入 `/stop` 中断流程
